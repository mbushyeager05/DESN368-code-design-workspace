Why reset at all? (What problem does it solve?)

Historically, the main goal of a CSS reset has been to ensure consistency between browsers, and to undo all default styles, creating a blank slate.

What's a CSS boilerplate? (vs. a reset)

A CSS boilerplate is the baseplate for the CSS. This includes the reset and normalized stylesheet. It also includes base typography. Such as fonts, wights, and size. This can also include things such as line height. Has the basic layout structure. Such as for grid or containers. The last thing that can be included in the CSS boilerplate is utility fixes and maybe common components.CSS reset is where either removes/normalize browser defaults.

Why use a boilerplate? (When does it help/hurt?)

There are pros and cons to using a bolder plate.

The pro's

  • For efficiency: Boilerplates speed up the creation of new projects and documents by providing a starting point.
  • For consistency: Using a boilerplate ensures that standard information is presented consistently across multiple documents.
  • For a foundational structure: It provides a basic, pre-built framework for projects.
  • For clarity: In legal contracts, boilerplate clauses provide clarity on standard terms.
  • For learning: It can be a great educational tool for junior developers to learn the structure of a technology or framework without starting from scratch. 

The cons:

  • Over-engineering: You include CSS you'll never use, bloating file sizes.
  • Fighting the boilerplate: Spending time overriding opinionated defaults.
  • Learning hindrance: Beginners might not understand the code they're using.
  • Unique projects: Highly custom designs may not fit the boilerplate's assumptions.
